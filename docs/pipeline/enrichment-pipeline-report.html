R4
Tiered Contact Discovery
Uses the Parent Account List to prioritize systems by segment and dollar opportunity. Processes every facility within each target parent system. For each facility, runs 11 DH API executive searches (C-Suite, Pharmacy, Nursing, Quality, Safety, Informatics, Critical Care, Surgery, Endocrinology, Diabetes, Hospital Medicine), deduplicates results, then runs ICP matching to tag the 66 target titles.
Scope: Enterprise/Strategic 242 parent systems 4,251 facilities ~11 API calls/facility ThreadPoolExecutor(5)

Method

  1. Load Parent Account List CSV → filter to Segment = "Enterprise/Strategic", exclude Glytec clients
  2. Sort by Adj Total $ Opp descending (highest opportunity first)
  3. For each system, get all facilities from Facility List CSV via Highest Level Parent join
  4. Skip facilities already in executives table (resume support)
  5. For each facility, call icp_discover_hospital() which runs 11 filtered executive searches
  6. Store all executives in DB, run ICP matcher, tag matches with title/score/category

DH API Endpoints

POST /v1/hospitals/{def_id}/executives/search   (11× per facility, one per ICP search batch)
  Batch filters:
    C-Suite:       positionLevel Equals "C-Level"
    Pharmacy:      department Contains "Pharm"
    Nursing:       department Contains "Nurs"
    Quality:       department Contains "Quality"
    Safety:        department Contains "Safety"
    Informatics:   department Contains "Informati"
    Critical Care: department Contains "Critical"
    Surgery:       department Contains "Surg"
    Endocrinology: standardizedTitle Contains "Endocrin"
    Diabetes:      standardizedTitle Contains "Diabetes"
    Hospital Med:  standardizedTitle Contains "Hospital"

R3
Network Leadership Change Detection
Scans every facility in target parent systems for recently updated executives (since Dec 2025). Leadership changes = trigger signals for outreach timing. Also pulls news articles about leadership, M&A, EHR migrations, new facilities, and financial distress.
Scope: Enterprise/Strategic 4,251 facilities 301,510 signals found ThreadPoolExecutor(5)

Method

  1. Get all target parent system names from Parent Account List CSV
  2. For each system, get all facilities from Facility List CSV
  3. Skip facilities already in trigger_signals table (resume support)
  4. For each facility, search executives with lastUpdatedDate > 2025-12-01
  5. Store each recently-updated executive as a "leadership_change" trigger signal

DH API Endpoints

R6
Financial Distress + Quality Penalty Contact Discovery
Pre-filters from CSV for hospitals with negative operating margins AND ICU beds (pain + relevance). Enriches with CMS HAC (Hospital-Acquired Condition) penalties from the API, then pulls ICP-targeted contacts. These hospitals have the most urgent financial reason to adopt Glucommander.
Scope: Enterprise/Strategic 1,189 distressed facilities 2,993 contacts 1,819 quality penalties ThreadPoolExecutor(5)

Method

  1. Filter Facility List CSV: Net Operating Profit Margin < 0% AND ICU Beds > 0
  2. Filter to Enterprise/Strategic parent systems (exclude Glytec clients)
  3. For each distressed facility: search CMS quality metrics for HAC penalties
  4. Store penalty records in account_enrichment table (type = "quality_penalty")
  5. Pull ICP-targeted contacts via icp_discover_hospital()

DH API Endpoints

POST /v1/hospitals/{def_id}/quality/search
  Filter: name Contains "hacPenalty"

POST /v1/hospitals/{def_id}/executives/search   (11× ICP batches)

Fields Collected

Executives → executives table (same as R4). Quality penalties → account_enrichment table: data_type, metric_name, metric_value, numeric_value, year, description.

R8
Physician Champion Pipeline
Finds endocrinologists, hospitalists, internists, and medical directors who could champion Glucommander adoption within their hospitals. Filters to SAC (Short Term Acute Care) hospitals with ≥10 ICU beds. Also checks the executive endpoint for physician leaders.
Scope: Enterprise/Strategic 1,387 ICU facilities 252,016 physicians 4 specialty searches ThreadPoolExecutor(5)

Method

  1. Filter Facility List: Hospital Type = "Short Term Acute Care Hospital" AND ICU Beds ≥ 10
  2. Filter to Enterprise/Strategic parent systems (exclude Glytec clients)
  3. For each facility, run 4 physician specialty searches + 1 physician leader search
  4. Store all physicians in physicians table

R9
Comprehensive Account Enrichment Bundle
Pulls quality metrics, financial data, memberships (GPO, ACO, HIE), and news/trigger signals for every target hospital. This is the "account intelligence" layer that doesn't find contacts but enriches each hospital record with data that informs messaging and prioritization.
Scope: Enterprise/Strategic 4,249 facilities 105,131 enrichment records 4 API calls/facility ThreadPoolExecutor(5)

Method

  1. Filter to Enterprise/Strategic parent systems (exclude Glytec clients)
  2. For each facility, pull 4 categories of enrichment data from DH API
  3. Filter quality metrics to key names (HAC, readmission, VBP)
  4. Filter financial metrics to key names (margin, beds, discharges, IT budget, cash on hand)
  5. Pull all memberships (GPO, ACO, CIN, HIE, Payor)
  6. Search news for high-value types since June 2025

GET /v1/hospitals/{def_id}/memberships                 → GPO, ACO, CIN, HIE memberships
POST /v1/hospitals/{def_id}/newsandintelligence/search → news filtered to post-June 2025

Matching Algorithm
1. Load 66 ICP titles from ICP.json
2. Add 2 synthetic titles:
   - Chief Medical Officer (90, Executive)
   - Chief of Medicine (80, Executive)
3. For each DH executive:
   a. Keyword match on standardizedTitle (+60)
   b. Title word overlap vs ICP title (+40)
   c. Department match bonus (+20)
   d. Position level boost (+5-10)
   e. Score ≥ 50 → match
4. Explicitly reject non-ICP titles:
   CIO, CMO→CTO, Chief of Radiology,
   Pathology, Anesthesia, Emergency,
   Pediatrics, Psychiatry, Neurology,
   Chief of Staff, CHRO, CLO, CXO
